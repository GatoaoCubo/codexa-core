## 1. A CIÊNCIA DA RECIPROCIDADE

### 1.1 O Princípio (Cialdini, 1984)

A **Lei da Reciprocidade** é uma das 6 armas de persuasão identificadas por Robert Cialdini:

> "Quando alguém nos dá algo, sentimos uma obrigação interna de retribuir."

**Por que funciona**:
- É uma norma social universal (todas as culturas)
- Opera no nível subconsciente
- A retribuição geralmente EXCEDE o valor recebido
- Funciona mesmo quando o presente não foi solicitado

### 1.2 Aplicação CODEXA

```yaml
o_que_damos:
  - Video promo: Promessa de transformação (isca)
  - Video LP: Confirmação que a promessa é real
  - Módulo 0: Conhecimento aplicável HOJE (10 min)
  - Quick Wins: 3 prompts de alto valor (copy-paste)

o_que_recebemos:
  - Atenção: Tempo do usuário
  - Confiança: Credibilidade construída
  - Email/Lead: Acesso para comunicação
  - Conversão: Assinatura codexa.app
  - Advocacia: Indicações orgânicas
```

### 1.3 O Segredo: Valor REAL

A reciprocidade SÓ funciona se o presente for **genuinamente valioso**.

```
❌ ERRADO: Dar conteúdo raso → Usuário sente que foi enganado → Desconfiança

✅ CERTO: Dar conteúdo valioso → Usuário aplica e funciona → "Se o grátis é assim, imagina o pago"
```

## 2. ANATOMIA DO FUNIL

### 2.1 Estágio 1: VIDEO PROMO (Isca)

**Objetivo**: Capturar atenção e criar curiosidade

```markdown
PROMESSAS FEITAS:
- "De 2 horas para 5 minutos"
- "Sistema que trabalha sozinho"
- "Conhecimento proprietário"
- "Saia da Camada 1"

GATILHO: Curiosidade + Contraste
EMOÇÃO: "Isso é possível?"
```

**Risco**: Parecer mais uma promessa vazia de internet

### 2.2 Estágio 2: VIDEO LP (Confirmação)

**Objetivo**: Confirmar que as iscas NÃO eram fake

```markdown
ABERTURA CRÍTICA:
"Obrigado por confiar em nós." [take de 1 segundo]

POR QUE FUNCIONA:
- Reconhece que o usuário ESCOLHEU confiar
- Cria compromisso implícito ("agora preciso entregar")
- Diferencia de outros que só prometem

CONTEÚDO: Mostra que o sistema é real
- Demo ao vivo (não slides)
- Resultados verificáveis
- Sem hype, só fatos
```

**Transição**: Da desconfiança para "vou dar uma chance"

### 2.3 Estágio 3: MÓDULO 0 (Valor Real)

**Objetivo**: Entregar valor aplicável HOJE

```markdown
DURAÇÃO: 10 minutos (respeitando tempo do usuário)

CONTEÚDO:
1. Conceito: Destilação de Conhecimento
2. Framework: Os 4 CORES
3. Prática: 3 Quick Wins

RESULTADO TANGÍVEL:
- Usuário executa QW1, QW2, QW3
- Cria primeiro ativo de conhecimento
- Experimenta a transformação prometida

GATILHO ATIVADO: Reciprocidade
"Recebi isso de graça e FUNCIONOU. O que mais tem?"
```

### 2.4 Estágio 4: CODEXA.APP (Retribuição)

**Objetivo**: Converter gratidão em ação

```markdown
OFERTA:
- Curso completo (6 módulos)
- 6 agentes especializados
- Base de conhecimento e-commerce BR
- Acesso antecipado a novos features
- Descontos exclusivos para assinantes

POSICIONAMENTO:
Não é "compre meu produto"
É "se você quer ir além, aqui está o caminho"

DECISÃO DO USUÁRIO:
"Esse conteúdo gratuito me ajudou mais que cursos pagos.
Quero retribuir E continuar aprendendo."
```

## 3. OS QUICK WINS: CIRURGIA ESTRATÉGICA

### 3.1 O Dilema

```
Como dar algo valioso o suficiente para gerar reciprocidade
SEM depreciar a necessidade do produto completo?
```

### 3.2 A Solução

Os Quick Wins são **portas de entrada**, não substitutos.

```yaml
QW1_META_PROMPT:
  valor: "Aprenda a delegar para IA"
  limitacao: "Você ainda faz tudo manual, um por um"
  upgrade_codexa: "Agentes fazem isso automaticamente em escala"

QW2_PLAN:
  valor: "Vire supervisor, não executor"
  limitacao: "Você valida, mas não orquestra"
  upgrade_codexa: "Workflows orquestram múltiplos agentes"

QW3_IMPLEMENT:
  valor: "Crie conhecimento portátil"
  limitacao: "Arquivos soltos, sem estrutura"
  upgrade_codexa: "ISO Vectorstore organiza em sistema"
```

### 3.3 A Metáfora

```
Quick Wins = Amostra grátis do restaurante

Se a amostra for ruim → Você não entra
Se a amostra for boa → Você quer o prato completo

CODEXA = O prato completo
```
